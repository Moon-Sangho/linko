# Linko — v0.2 Scope

_Written by: /agent-pm | Last updated: 2026-03-17_

---

## v0.2 Theme: "Organize"

If v0.1 completed the workflow for *saving* bookmarks,
v0.2 completes the workflow for **managing and organizing** saved data.

---

## v0.2 Feature List

| Feature | Requirement IDs | User Story | Notes |
|---------|----------------|------------|-------|
| Tag rename | TG-4 | US-07 | Reflected immediately across all bookmarks |
| Tag delete | TG-5 | US-07 | Bookmarks preserved; tag detached only |
| Tag bookmark count display | TG-5 (display) | US-07 | Shows bookmark count next to each tag |
| Bulk delete | BM-8 | US-14 | Select mode + confirm dialog |
| Export JSON | EX-1 | US-11 | All bookmarks + tags included |

---

## Schema Changes

v0.2 makes no changes to the existing schema.

- **Tag rename/delete**: Handled by existing `tags` + `bookmark_tags` join table
- **Bulk delete**: Existing `bookmarks` table; only adds multi-id handling
- **JSON export**: Read-only; no schema changes required

No migration needed.

## New IPC Channels (v0.2)

| Channel | Handler | Notes |
|---------|---------|-------|

| `tag:delete` | `src/main/ipc/tags.ts` | `{ id }` → `IpcResult<void>` |
| `bookmark:delete-bulk` | `src/main/ipc/bookmarks.ts` | `{ ids: number[] }` → `IpcResult<void>` |
| `bookmark:export-json` | `src/main/ipc/bookmarks.ts` | `{}` → native save dialog + write |

## Success Criteria

- A mistakenly named tag can be corrected without reinstalling the app
- Among 500 bookmarks imported via browser HTML, unwanted ones can be cleaned up in under 10 clicks
- A file generated by JSON export parses successfully via `JSON.parse()`
